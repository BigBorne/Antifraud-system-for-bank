## Сервис антифрода
#### Легенда
Сервис принимает набор данных клиента, содержащихся в сторонней системе (хранилище данных). Затем, на основе переданных данных осуществляет проверки правил, препятствующих дальнейшему кредитному скорингу.

#### Работа сервиса
- сервис реализует несколько эндпоинтов
- входные данные эндпоинтов: дата рождения, номер телефона, история займов (список из объектов, каждый из которых содержит сумму займа, дату его оформления, и флаг закрытия), блокировка/разблокировка юзеров
- логика эндпоинта /check:
    - сервис делает проверки:
      - если телефон начинает не с +7 (или 8), считаем, что сработал стоп-фактор
      - если клиенту меньше 18 лет, считаем, что сработал стоп-фактор
      - если у клиента есть хотя бы один не закрытый займ, считаем, что сработал стоп-фактор
      - если клиент находится в черном списке (блоке), считаем, что сработал стоп-фактор
- логика эндпоинта /admin/blacklist/add:
  - сервис добавляет клиента по номеру телефона в черный список
- логика эндпоинта /admin/blacklist/remove/{phone}:
  - сервис удаляет клиента по номеру телефона из черного списка
- в качестве ответа на эндпоинте сервис возвращает список из не пройденных проверок и итоговый флаг прохождения/не прохождения проверок
- сервис кэширует полученный результат в Redis

#### Примеры запроса/ответа
```json
// Запрос
{
  "birth_date": "2000-12-12",
  "phone_number": "89234535452",
  "loans_history": [
    {
      "amount": 100413,
      "loan_data": "2023-12-12",
      "is_closed": true
    },
{
      "amount": 123413,
      "loan_data": "2023-12-12",
      "is_closed": false
    }
  ]
}

// Ответ
{
  "message": "Антифрод система работает!",
  "stop_factors": [
    "Есть открытый кредит на сумму 123413"
  ],
  "passed": false
}
```
